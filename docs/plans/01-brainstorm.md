# 01 — Brainstorm (영어 발음 듣기 보드)

> 원격 무인 세션 → 사용자 질의 없이 brainstorming skill의 원칙(YAGNI, 1번에 1결정, 트레이드오프 명시)을 AI 판단으로 적용.

## 핵심 사용자 흐름 (교사 관점)
1. 수업 중 TV 큰 화면에 띄움
2. 새 단어/문장 입력 → 한 번에 듣기
3. 자주 쓰는 표현을 즐겨찾기에 박아두고 클릭 한 번으로 재생
4. 학생이 따라할 수 있는 속도로 슬라이더 조절
5. 단어 카드로 인쇄해 종이로 배포

## MUST (꼭 필요한 기능, 3+)
1. **TTS 즉시 재생** — 입력창에 단어/문장 입력 → 엔터 또는 ▶ 버튼 → 영어 음성 재생. 한국어 voice 자동 회피.
2. **즐겨찾기 보드** — 한 번 재생한 단어는 보드에 카드로 누적. 카드 ★ 표시로 즐겨찾기 토글. localStorage 영속.
3. **속도 슬라이더 (0.5–1.5x)** — 큰 슬라이더로 TV에서도 보임. 현재 배율 옆 숫자로 표기.
4. **카테고리 시드 데이터** — 빈 화면 회피. Greeting / Classroom / Animals / Food / Colors / Phrases 6개 카테고리에 미리 채워둔 단어.
5. **단어 카드 인쇄** — 즐겨찾기만 출력. A4 4×4 = 16장/페이지. `@media print`로 UI는 숨김.

## SHOULD (있으면 좋은 기능, 2+)
1. **voice 선택 드롭다운** — `en-US`, `en-GB`, `en-AU` 등 가용 voice 표시 (브라우저 환경별 상이). 미선택 시 자동 우선순위.
2. **클릭 시 시각 피드백** — 재생 중인 카드 가장자리에 ElevenLabs 시그니처인 atmospheric gradient orb 펄스 (mint→peach 드리프트). 음성 AI 브랜드 정렬 + 청각 ↔ 시각 동기화.

## MUST NOT (절대 금지, 2+, scope creep 방지)
1. **학생 음성 녹음·분석 / AI 발음 평가** — 토픽 명시 배제. 마이크 권한 자체 요청 안 함. 학생 개인정보 보호.
2. **계정·로그인·서버 동기화** — 100% 클라이언트 로컬. 백엔드 없음. 학교 IT 부서 승인 절차 회피.
3. **외부 단어집 API 호출 / 사전 정의 lookup** — Web Speech API만으로 충분. 외부 의존 0 원칙.
4. **다국어 (한국어/일본어/중국어) TTS** — 토픽은 영어 발음. 다국어 추가는 voice 우선순위 로직 복잡도만 증가시키고 교사 사용 시나리오에 불필요.

## 트레이드오프 / 결정 사항

### 결정 1: 카테고리 탭 vs 평면 리스트
- **선택**: 카테고리 탭 (Greeting/Classroom/.../Favorites 7개).
- **이유**: 30개+ 시드 + 즐겨찾기까지 평면 리스트면 스크롤이 길어짐. 카테고리 탭이면 한 화면에 다 보임. TV 시청에 더 적합.
- **트레이드오프**: 탭 UI 코드 증가, 하지만 vanilla JS 50줄 이내.

### 결정 2: 입력 → 자동 카테고리 분류?
- **선택**: 분류 안 함. 즉석 입력은 "최근 입력" 가상 카테고리에 누적. 즐겨찾기는 ★ 누른 카드만.
- **이유**: 자동 분류는 LLM 없으면 부정확, LLM 쓰면 토픽 제약(AI: ✕) 위반.

### 결정 3: 인쇄 레이아웃
- **선택**: 즐겨찾기만 인쇄. 4×4 = 페이지당 16장. 단어 + 한글 발음 표기는 **표시하지 않음** (교사가 발음을 들려주는 보조 도구 ↔ 카드는 단어 인식용).
- **이유**: 한국식 발음 표기를 인쇄하면 토픽 취지(영어 그대로 듣기) 훼손.

## 비-기능 요구
- **접근성**: 색 대비 4.5:1 (ElevenLabs 색 토큰 검증 필요), 모든 버튼 `aria-label`, 키보드 포커스 ring, ▶ 버튼은 Space 키 토글.
- **반응형**: TV(1920×1080) / 데스크탑(1280) / 태블릿(768) 3-tier. 모바일은 보조.
- **무서버**: 100% 정적, GitHub Pages 직접 서빙 가능.
